## 🚀 缓存系统性能分析

### L1 缓存（完整结果缓存）

**工作原理**:
```
缓存键 = MD5(company_id | normalized_query | response_mode)
存储内容 = {
    "result": {完整管道结果},
    "display_data": {格式化显示数据},
    "interpretation": {LLM 生成的解释},
    "route": {路由信息}
}
```

**性能提升**:
- Quick 模式 + L1 命中 → **0 LLM 调用**（<100ms）
- Think 模式 + L1 命中 → **1 LLM 调用**（仅重新解释，~2-3s）
- Deep 模式 → 绕过缓存，完整执行（~5-10s）

### L2 缓存（SQL 模板缓存）

**域感知缓存键**:
```
财务报表: MD5(query | mode | fs | accounting_standard)
增值税:   MD5(query | mode | vat | taxpayer_type)
企业所得税: MD5(query | mode | eit)
```

**智能适配**:
- 财务报表自动切换 `_eas` ↔ `_sas` 视图
- 跨公司复用 SQL 模板
- 仅需实例化 `{taxpayer_id}` 占位符

**性能提升**:
- L1 未命中 + L2 命中 → **跳过 Stage 2**（节省 1 次 LLM 调用，~3-5s）

### 实际性能对比

| 场景 | LLM 调用 | 响应时间 | 节省 | 适用模式 |
|------|---------|---------|------|----------|
| 无缓存 | 2 次 | ~5-10 秒 | - | Deep |
| L2 命中 | 1 次 | ~3-5 秒 | 50% | Deep |
| L1 命中 | 0 次 | <100 毫秒 | 100% | Quick |
| L1 命中 + 重新解释 | 1 次 | ~2-3 秒 | 50% | Think |
