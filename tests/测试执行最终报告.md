## 🚀 缓存系统性能

### L1 缓存（完整结果缓存）

**工作原理**:
- 缓存键: `MD5(company_id|normalized_query|response_mode)`
- 存储内容: 完整的管道结果（SQL、结果、display_data、interpretation）
- 持久化: 文件系统（`cache/` 目录）
- LRU 淘汰: 最大 1500 个文件

**性能提升**:
- Quick 模式 + L1 命中 → **0 LLM 调用**（即时返回）
- Think 模式 + L1 命中 → **1 LLM 调用**（仅重新解释）
- Deep 模式 → 绕过缓存，完整执行

### L2 缓存（SQL 模板缓存）

**工作原理**:
- 域感知缓存键:
  - 财务报表: `MD5(query|mode|fs|accounting_standard)`
  - 增值税: `MD5(query|mode|vat|taxpayer_type)`
  - 企业所得税: `MD5(query|mode|eit)`
- 存储内容: SQL 模板（带 `{taxpayer_id}` 占位符）
- 智能适配: 财务报表自动切换 `_eas` ↔ `_sas` 视图

**性能提升**:
- L1 未命中 + L2 命中 → **跳过 Stage 2**（节省 1 次 LLM 调用）
- 跨公司复用: 同一查询类型在不同公司间复用模板

### 实际效果

| 场景 | LLM 调用次数 | 响应时间 | 节省 |
|------|-------------|---------|------|
| 无缓存 | 2 次（Stage 1 + Stage 2） | ~5-10 秒 | - |
| L2 命中 | 1 次（仅 Stage 1） | ~3-5 秒 | 50% |
| L1 命中（Quick） | 0 次 | <100 毫秒 | 100% |
| L1 命中（Think） | 1 次（仅解释） | ~2-3 秒 | 50% |
