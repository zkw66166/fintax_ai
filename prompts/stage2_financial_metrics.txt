【跨期比较查询模式】★ 当用户问"比较"/"对比"/"变化"时必须使用此模式 ★
注意：此模式仅用于明确的"比较"/"对比"查询（如"比较2024与2025年末资产负债率"）。
如果用户问"从X到Y"/"X至Y月"等范围查询，应使用【趋势查询模式】，不要用此模式。

当用户比较同一指标在不同期间的数据时，使用UNION ALL模式返回所有有数据的期间。
重要：使用UNION ALL而不是JOIN，这样即使某个期间数据缺失，其他期间的数据仍会返回。

WITH base AS (
  SELECT
    period_year,
    period_month,
    metric_name,
    metric_value,
    metric_unit,
    evaluation_level
  FROM vw_financial_metrics
  WHERE taxpayer_id = :taxpayer_id
    AND ((period_year = 2024 AND period_month = 12) OR (period_year = 2025 AND period_month = 12))
    AND period_type = 'annual'
    AND metric_name IN ('资产负债率', '净利率')
)
SELECT
  metric_name,
  MAX(CASE WHEN period_year = 2024 AND period_month = 12 THEN metric_value END) AS "2024年末",
  MAX(CASE WHEN period_year = 2025 AND period_month = 12 THEN metric_value END) AS "2025年末",
  ROUND(
    MAX(CASE WHEN period_year = 2025 AND period_month = 12 THEN metric_value END) -
    MAX(CASE WHEN period_year = 2024 AND period_month = 12 THEN metric_value END),
    2
  ) AS "变动",
  MAX(metric_unit) AS metric_unit
FROM base
GROUP BY metric_name
LIMIT {max_rows}

要点：
- 使用CASE WHEN透视，每个期间一列，列别名用中文标注期间（如 "2024年末"）
- 变动额列：后期 - 前期（如果某期数据缺失，变动额为NULL）
- 即使某个期间数据不存在，其他期间的数据仍会显示（NULL表示数据缺失）
- 禁止用 BETWEEN 返回中间所有月份，只取起止两个期间
- "年末"对应 period_month = 12, period_type = 'annual'

【跨期比较 - 不同月份类型示例】
当比较季末月与非季末月时，每个CTE独立选择period_type：
WITH base AS (
  SELECT
    period_year,
    period_month,
    metric_name,
    metric_value,
    metric_unit
  FROM vw_financial_metrics
  WHERE taxpayer_id = :taxpayer_id
    AND ((period_year = 2025 AND period_month = 12 AND period_type IN ('quarterly', 'annual'))
         OR (period_year = 2026 AND period_month = 3 AND period_type IN ('quarterly', 'annual')))
    AND metric_name IN ('净利率', '增值税税负率', '企业所得税税负率')
)
SELECT
  metric_name,
  MAX(CASE WHEN period_year = 2025 AND period_month = 12 THEN metric_value END) AS "2025年12月",
  MAX(CASE WHEN period_year = 2026 AND period_month = 3 THEN metric_value END) AS "2026年3月",
  ROUND(
    MAX(CASE WHEN period_year = 2026 AND period_month = 3 THEN metric_value END) -
    MAX(CASE WHEN period_year = 2025 AND period_month = 12 THEN metric_value END),
    2
  ) AS "变动",
  MAX(metric_unit) AS metric_unit
FROM base
GROUP BY metric_name
LIMIT {max_rows}

【分类汇总明细查询模式】★ 当用户问"总和"/"合计"/"列出所有"时必须使用此模式 ★
注意：SQLite的UNION ALL不支持ORDER BY中使用CASE表达式，必须用sort_key列排序。
SELECT metric_name, metric_value, metric_unit, evaluation_level FROM (
  SELECT metric_name, metric_value, metric_unit, evaluation_level, 0 AS sort_key
  FROM vw_financial_metrics
  WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
    AND metric_category = '盈利能力'
  UNION ALL
  SELECT '合计' AS metric_name, SUM(metric_value) AS metric_value, '' AS metric_unit, NULL AS evaluation_level, 1 AS sort_key
  FROM vw_financial_metrics
  WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
    AND metric_category = '盈利能力'
)
ORDER BY sort_key, metric_name
LIMIT {max_rows}

【趋势查询模式】★ 当用户问"从X到Y"/"X至Y月"等范围查询，或"趋势"/"走势"/"逐月"/"各月"时必须使用此模式 ★
当用户查询某指标多个月/季度/年的变化趋势时，每个期间一行：
SELECT period_year, period_month, period_type, metric_name, metric_value, metric_unit, evaluation_level
FROM vw_financial_metrics
WHERE taxpayer_id = :taxpayer_id
  AND (period_year*100+period_month) BETWEEN :start_yyyymm AND :end_yyyymm
  AND metric_name = '资产负债率'
  AND period_type IN ('quarterly', 'annual')
  AND metric_value IS NOT NULL
ORDER BY period_year, period_month
LIMIT {max_rows}

注意：
- 范围查询（"X至Y"/"X到Y"/"从X到Y"）必须用此模式，不要用跨期比较模式
- 必须加 metric_value IS NOT NULL 过滤，避免返回空值行
- 必须加 period_type IN ('quarterly', 'annual') 过滤，避免月度空值
- 返回范围内所有有数据的期间，不要只返回起止两个端点
- 对于企业所得税税负率等可能为NULL的指标，JOIN时会自动过滤

【常见查询模式】
1. 查询某个指标：WHERE metric_name = '毛利率' 或 WHERE metric_code = 'gross_margin'
2. 查询某类指标：WHERE metric_category = '盈利能力'
3. 查询全部指标：不加 metric_name/metric_code/metric_category 过滤
4. 评价筛选：WHERE evaluation_level = '优'
5. 按粒度查询：WHERE period_type = 'monthly' / 'quarterly' / 'annual'
6. 查询年度指标：WHERE period_type = 'annual' AND period_year = :year AND period_month = 12
7. 查询季度指标：WHERE period_type = 'quarterly' AND period_year = :year AND period_month = :quarter_end_month
