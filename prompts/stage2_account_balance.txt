【跨期比较查询模式】★ 当用户问"比较"/"对比"/"变化"时必须使用此模式 ★
当用户比较同一科目在不同期间的数据时（如"比较应收账款在2023年12月与2024年12月的余额变化"），
使用两个CTE分别查询两个期间，JOIN输出对比列+变动额。禁止使用BETWEEN返回中间月份。
WITH t1 AS (
  SELECT account_code, account_name, closing_balance
  FROM vw_account_balance
  WHERE taxpayer_id = :taxpayer_id AND period_year = 2023 AND period_month = 12
    AND account_name = '应收账款'
  ORDER BY revision_no DESC LIMIT 1
),
t2 AS (
  SELECT account_code, account_name, closing_balance
  FROM vw_account_balance
  WHERE taxpayer_id = :taxpayer_id AND period_year = 2024 AND period_month = 12
    AND account_name = '应收账款'
  ORDER BY revision_no DESC LIMIT 1
)
SELECT t1.account_name AS account_name,
       t1.closing_balance AS "2023年12月余额",
       t2.closing_balance AS "2024年12月余额",
       (t2.closing_balance - t1.closing_balance) AS "变动额"
FROM t1 JOIN t2 ON t1.account_code = t2.account_code
LIMIT {max_rows}

要点：
- 每个期间一列，列别名用中文标注期间（如 "2023年12月余额"）
- 最后一列为变动额（后期 - 前期）
- 如果比较多个科目，每个科目一行
- 禁止用 BETWEEN 返回中间所有月份，只取起止两个月

【分类汇总明细查询模式】★ 当用户问"总和"/"合计"/"列出所有"时必须使用此模式 ★
当用户要求"列出XX类科目的XX总和"时（如"列出所有资产类科目期初余额总和"），
必须先列出每个科目的明细行，最后 UNION ALL 一行合计。禁止只返回SUM。
注意：SQLite的UNION ALL不支持ORDER BY中使用CASE表达式，必须用sort_key列排序。
WITH latest AS (
  SELECT account_code, account_name, opening_balance,
    ROW_NUMBER() OVER (PARTITION BY account_code ORDER BY revision_no DESC) AS rn
  FROM vw_account_balance
  WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
    AND account_code LIKE '1%' AND level = 1
)
SELECT account_code, account_name, opening_balance FROM (
  SELECT account_code, account_name, opening_balance, 0 AS sort_key
  FROM latest WHERE rn = 1
  UNION ALL
  SELECT '合计' AS account_code, '合计' AS account_name, SUM(opening_balance) AS opening_balance, 1 AS sort_key
  FROM latest WHERE rn = 1
)
ORDER BY sort_key, account_code
LIMIT {max_rows}

要点：
- 明细行在前，合计行在后
- 合计行的维度列填"合计"
- 按 account_code 排序，合计行排最后
- 按科目编号筛选用 account_code LIKE '1%'（资产类）、'2%'（负债类）、'3%'（权益类）、'4%'（成本类）、'5%'（损益类）

【多科目查询模式】
当用户查询多个科目（如"查询银行存款和应收账款的期末余额"），必须每个科目一行：
SELECT account_code, account_name, closing_balance
FROM vw_account_balance
WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
  AND account_name IN ('银行存款', '应收账款')
ORDER BY account_code
LIMIT {max_rows}

【趋势查询模式】
当用户查询某科目多个月的变化趋势时，每个月一行：
SELECT period_year, period_month, account_name, closing_balance
FROM vw_account_balance
WHERE taxpayer_id = :taxpayer_id
  AND (period_year*100+period_month) BETWEEN :start_yyyymm AND :end_yyyymm
  AND account_name = '应收账款'
ORDER BY period_year, period_month
LIMIT {max_rows}
