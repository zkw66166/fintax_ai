【必备过滤】每个子查询都必须包含：

【跨域SQL生成策略】
- 使用 UNION ALL 将不同视图的查询结果合并，每个子查询输出统一的列结构
- 统一输出格式：taxpayer_name, domain_label, metric_name, metric_value
- domain_label 用中文标识数据来源（如 '增值税', '利润表', '资产负债表' 等）
- 每个子查询只查询该视图中存在的列
- 严格禁止跨视图JOIN（不同域的视图之间不能JOIN）
- 严格禁止 SELECT *

【示例：利润表+增值税跨域查询】
WITH profit_data AS (
  SELECT taxpayer_name, '利润表' AS domain_label,
         'operating_revenue' AS metric_name,
         CAST(operating_revenue AS TEXT) AS metric_value
  FROM vw_profit_eas
  WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
    AND time_range = '本年累计'
  LIMIT 1
),
vat_data AS (
  SELECT taxpayer_name, '增值税' AS domain_label,
         'sales_taxable_rate' AS metric_name,
         CAST(sales_taxable_rate AS TEXT) AS metric_value
  FROM vw_vat_return_general
  WHERE taxpayer_id = :taxpayer_id AND period_year = :year AND period_month = :month
    AND item_type = '一般项目' AND time_range = '累计'
  LIMIT 1
)
SELECT * FROM profit_data
UNION ALL
SELECT * FROM vat_data
LIMIT {max_rows}

【跨年季度比较查询】
PHASE 2 FIX: 当【用户意图JSON】中出现 cross_year_compare 字段且 is_quarter_query=true 时，表示用户要比较两个不同年份的同一季度。
例如 cross_year_compare.year1=2024, cross_year_compare.year2=2025, cross_year_compare.quarter=1, cross_year_compare.months_in_quarter=[1,2,3]

此时必须协调多个子域的SQL模板：
- 每个子域的SQL都应该使用 period_year IN (:year1, :year2) AND period_month = :month1 过滤
- 季度末月份（如Q1=3月）作为 :month1 参数
- 利润表/现金流量表：使用 time_range = '本年累计' 获取季度累计数据

- 增值税：使用季度末月份的累计申报数据
- 企业所得税：如果是季报，使用 period_quarter = :quarter 过滤

【财务指标域特殊处理】
对于 financial_metrics 域 (vw_financial_metrics 视图):

**数据模型**:
- 该视图使用 metric_name 列存储指标名称（中文）
- 指标值存储在 metric_value 列（REAL类型）
- 指标单位存储在 metric_unit 列（如"%"、"元"、"次"）
- 评估等级存储在 evaluation_level 列（如"优秀"、"良好"、"一般"、"较差"）

**查询方式**:
当 select.metrics 或 select.user_intent_metrics 包含含"率"的中文指标名称时：
1. 使用 WHERE metric_name = '指标中文名称' 过滤
2. SELECT 必须包含: metric_name, metric_value, metric_unit, evaluation_level
3. 不要尝试将中文指标名映射到物理列名（它们本身就是 metric_name 的值）

**示例**:
- user_intent_metrics 包含 "所得税税负率"
  → WHERE metric_name = '所得税税负率'
- user_intent_metrics 包含 "增值税税负率"
  → WHERE metric_name = '增值税税负率'
- user_intent_metrics 包含 ["所得税税负率", "增值税税负率"]
  → WHERE metric_name IN ('所得税税负率', '增值税税负率')

**常见财务指标**:
- 税负类: 企业所得税税负率、增值税税负率、综合税负率
- 盈利类: 销售毛利率、销售净利率、净资产收益率(ROE)、总资产收益率(ROA)
- 偿债类: 资产负债率、流动比率、速动比率
- 营运类: 应收账款周转率、存货周转率、总资产周转率
- 成长类: 营业收入增长率、净利润增长率、总资产增长率
