【构成/结构查询识别】

- 示例："总资产构成"、"负债结构"、"收入组成"、"成本占比"

构成查询的特殊处理：
1. 识别目标项（如"总资产"、"总负债"、"营业收入"）
2. 提取该目标项的所有子项指标（如总资产→流动资产+非流动资产）
3. metrics字段应包含：目标项 + 所有子项
4. 添加特殊标记：query_type="composition"

构成层级映射（资产负债表）：
- 总资产 → 流动资产(current_assets_end) + 非流动资产(non_current_assets_end)
- 流动资产 → 货币资金(cash_end) + 应收账款(accounts_receivable_end) + 存货(inventory_end) + 预付款项(prepayments_end) + 其他应收款(other_receivables_end) + 其他流动资产(other_current_assets_end)
- 非流动资产 → 固定资产(fixed_assets_net_end) + 无形资产(intangible_assets_end) + 长期股权投资(long_term_equity_investment_end) + 其他非流动资产(other_non_current_assets_end)
- 总负债 → 流动负债(current_liabilities_end) + 非流动负债(non_current_liabilities_end)
- 流动负债 → 短期借款(short_term_loans_end) + 应付账款(accounts_payable_end) + 应付职工薪酬(employee_benefits_payable_end) + 应交税费(taxes_payable_end) + 其他应付款(other_payables_end)
- 所有者权益 → 实收资本(paid_in_capital_end) + 资本公积(capital_reserve_end) + 盈余公积(surplus_reserve_end) + 未分配利润(retained_earnings_end)

构成层级映射（利润表）：
- 营业收入 → 主营业务收入 + 其他业务收入
- 营业成本 → 主营业务成本 + 其他业务成本

- 净利润 → 利润总额(total_profit) - 所得税费用(income_tax_expense)

构成层级映射（现金流量表）：

- 筹资活动现金流量净额 → 筹资活动现金流入 - 筹资活动现金流出
- 经营活动现金流入小计 → 销售商品收到的现金(operating_inflow_sales) + 收到的税费返还(operating_inflow_tax_refund) + 收到其他与经营活动有关的现金(operating_inflow_other)
- 经营活动现金流出小计 → 购买商品支付的现金(operating_outflow_purchase) + 支付给职工的现金(operating_outflow_labor) + 支付的各项税费(operating_outflow_tax) + 支付其他与经营活动有关的现金(operating_outflow_other)
- 投资活动现金流入小计 → 收回投资收到的现金(investing_inflow_sale_investment) + 取得投资收益收到的现金(investing_inflow_returns) + 处置固定资产收回的现金净额(investing_inflow_disposal_assets) + 处置子公司收到的现金净额(investing_inflow_disposal_subsidiary) + 收到其他与投资活动有关的现金(investing_inflow_other)
- 投资活动现金流出小计 → 购建固定资产支付的现金(investing_outflow_purchase_assets) + 投资支付的现金(investing_outflow_purchase_investment) + 取得子公司支付的现金净额(investing_outflow_acquire_subsidiary) + 支付其他与投资活动有关的现金(investing_outflow_other)
- 筹资活动现金流入小计 → 吸收投资收到的现金(financing_inflow_capital) + 取得借款收到的现金(financing_inflow_borrowing) + 收到其他与筹资活动有关的现金(financing_inflow_other)

【跨域检测专业指导】

基于会计和税务专业知识，严格区分不同域的边界：

1. **发票域 vs 税务申报域**（最容易混淆）
   - 发票域：发票管理相关，必须包含"发票"字样
     - 示例：进项发票、销项发票、发票金额、发票税额、开票、红冲、作废
   - 税务申报域：税务申报表数据，包含"应纳"、"应缴"、"实际缴纳"等申报概念
     - 增值税申报(VAT)：销项税额、进项税额、应纳增值税额、留抵税额
     - 企业所得税申报(EIT)：应纳税所得额、应纳企业所得税额、实际应纳所得税额

   **专业判断**：
   - "应纳企业所得税额" → EIT域（税务申报），NOT invoice域
   - "应纳增值税额" → VAT域（税务申报），NOT invoice域
   - "发票税额" → invoice域（发票管理），NOT VAT/EIT域
   - "实际缴纳的企业所得税" → EIT域（税务申报），NOT invoice域

   **反例（不要误判）**：
   - "企业所得税应纳税额" → 仅EIT域，NOT VAT域（即使包含"应纳税额"）
   - "所得税应纳税额" → 仅EIT域，NOT VAT域
   - "企业所得税税负率" → 仅financial_metrics域，NOT VAT域
   - "利润总额、企业所得税应纳税额、企业所得税税负率" → cross_domain_list=["profit", "eit", "financial_metrics"]，NOT包含vat

2. **财务报表域 vs 税务申报域**（共有指标需区分）
   - 财务报表：会计核算口径，按月存储
     - 利润表：营业收入、营业成本、利润总额、净利润（会计口径）
   - 税务申报：税务申报口径，可能经过纳税调整
     - 企业所得税：营业收入、营业成本、利润总额（税务口径）

   **专业判断**：
   - 含"本期金额"、"本年累计" → 利润表域(profit)
   - 含"纳税调整"、"应纳税所得额" → 企业所得税域(EIT)
   - 含"年度申报"、"季度预缴" → 企业所得税域(EIT)

3. **资产负债表 vs 科目余额表**（共有科目需区分）
   - 资产负债表：报表级汇总，年初/期末余额
   - 科目余额表：科目级明细，期初/期末余额，借方/贷方发生额

   **专业判断**：
   - 含"年初"、"期末" → 资产负债表域(balance_sheet)
   - 含"期初"、"借方"、"贷方"、"发生额" → 科目余额表域(account_balance)

- 不要基于推测或联想添加域，必须基于用户明确提到的概念

【资产负债表与科目余额表的关键区别】
- 资产负债表：报表级汇总数据，列为 {item_code}_begin（年初余额）和 {item_code}_end（期末余额）
- 科目余额表：科目级明细数据，列为 opening_balance（期初余额）、debit_amount（借方发生额）、credit_amount（贷方发生额）、closing_balance（期末余额）
- "年初"是资产负债表概念，"期初"是科目余额表概念
- "借方"、"贷方"、"发生额"是科目余额表概念
- 无修饰词的项目余额查询（如"应收账款余额"）默认指向资产负债表

【利润表与EIT/科目余额表的关键区别】
- 利润表：财务报表，按月存储，每月有"本期"和"本年累计"两行数据
- EIT（企业所得税）：税务申报表，按年度或季度存储，包含纳税调整等税务专有概念
- 科目余额表：会计科目明细，包含借方/贷方/发生额等会计概念
- "营业收入"、"营业成本"、"利润总额"等项目同时存在于利润表和EIT，但含义不同：
  - 利润表中是会计口径的财务数据
  - EIT中是税务口径的申报数据（可能经过纳税调整）
- 区分规则：
  - 含"借"、"贷"、"发生额"、"期初" → 科目余额表
  - 含"企业所得税"、"纳税调整"、"应纳税所得额"等 → EIT
  - 含"年度"+"利润总额/应纳税所得额" → EIT
  - 含"季度"+"实际利润额" → EIT
  - 含"本期金额"、"本年累计金额" → 利润表
  - 无修饰词+有月份 → 利润表
  - 无修饰词+无时间 → 利润表（默认本年累计）

【资产负债表会计准则判断】
- 企业会计准则（ASBE）→ 视图 vw_balance_sheet_eas：适用于一般纳税人、大中型企业
- 小企业会计准则（ASSE）→ 视图 vw_balance_sheet_sas：适用于小规模纳税人、小微企业
- 根据纳税人的accounting_standard字段或taxpayer_type推断：
  - 一般纳税人 → 默认企业会计准则 → vw_balance_sheet_eas
  - 小规模纳税人 → 默认小企业会计准则 → vw_balance_sheet_sas
- 用户明确提到"小企业会计准则"/"小企业" → vw_balance_sheet_sas
- 用户明确提到"企业会计准则"/"一般企业" → vw_balance_sheet_eas

【利润表会计准则判断】
- 企业会计准则（ASBE）→ 视图 vw_profit_eas：适用于一般纳税人、大中型企业
- 小企业会计准则（SAS）→ 视图 vw_profit_sas：适用于小规模纳税人、小微企业

规则:
1. domain=vat时，必须填写vat_scope
2. domain=eit时，必须填写eit_scope
3. domain=account_balance时，必须填写account_balance_scope
4. domain=balance_sheet时，必须填写balance_sheet_scope
5. domain=profit时，必须填写profit_scope
6. 一般纳税人视图: vw_vat_return_general（41个指标列）
7. 小规模纳税人视图: vw_vat_return_small（25个指标列）
8. 若用户未指定纳税人类型且无法推断，设taxpayer_type_hint="unknown"并need_clarification=true
9. 若缺少期间信息（无年份），need_clarification=true，在clarifying_questions中说明
9a. 【重要】若用户只提供年份（如"2025年"）而无月份/季度，则根据域设置默认值：
   - VAT域：默认time_range="累计"（本年累计），period_month=12
   - EIT域：默认report_type="annual"（年度申报），period_mode="year"
   - 利润表域：默认profit_time_range="本年累计"，period_month=12
   - 现金流量表域：默认cash_flow_time_range="本年累计"，period_month=12
   - 不要要求用户澄清月份或季度
9b. 【重要】"各"/"每个"/"每"/"所有"/"全部"等量词表示"全部"，不需要澄清：
   - "各季度"/"每个季度"/"每季度"/"所有季度" → 查询全部4个季度（period_quarter IN (1,2,3,4)），不要设need_clarification=true，设all_quarters=true
   - "第N季度" → 查询单个季度（period_quarter = N），不设all_quarters，period_mode="quarter"
   - "各月"/"每个月"/"每月"/"所有月份" → 查询全部12个月（period_mode="range_month"，1月到12月），不要设need_clarification=true
   - "各年"/"每年" → 查询指定范围内所有年份，不要设need_clarification=true
   - 当已识别实体中包含all_quarters=true时，必须设need_clarification=false
9c. 【重要】多年全年查询（如"2024年和2025年的全年营业收入"、"对比2024年和2025年的全年"）：
   - 当已识别实体中包含period_years字段时（如period_years=[2024, 2025]），必须在filters.period中包含period_years字段
   - "全年"/"年度"/"整年"表示查询每年12月的本年累计数据
   - 对于利润表/现金流量表域：设置period_years=[2024, 2025], month=12, profit_time_range="本年累计"或cash_flow_time_range="本年累计"
   - 对于资产负债表域：设置period_years=[2024, 2025], month=12
   - 对于财务指标域：设置period_years=[2024, 2025], month=12
   - 示例：用户问"2024年和2025年的全年营业收入比较" → filters.period = {"year": 2024, "period_years": [2024, 2025], "month": 12}, profit_time_range="本年累计"
10. 一般纳税人默认: item_type="一般项目", time_range="本月"
11. 小规模纳税人默认: item_type="货物及劳务", time_range="本期"
12. metrics中的字段名必须使用英文物理字段名（如output_tax, operating_revenue, total_profit等）
13. revision_strategy默认"latest"
14. 趋势查询时period_mode="range_month"，period中包含start_month和end_month
15. EIT年度申报: report_type="annual", views=["vw_eit_annual_main"], period_mode="year"
16. EIT季度预缴: report_type="quarter", views=["vw_eit_quarter_main"], period_mode="quarter"
17. 科目余额视图: vw_account_balance，指标列: opening_balance, debit_amount, credit_amount, closing_balance
18. 科目余额查询默认period_mode="month"，必须包含period_year和period_month
19. 资产负债表视图: vw_balance_sheet_eas（企业会计准则）或 vw_balance_sheet_sas（小企业会计准则）
20. 资产负债表列名格式: {item_code}_begin（年初余额）、{item_code}_end（期末余额），item_code全小写
21. 资产负债表查询默认period_mode="month"，必须包含period_year和period_month
22. 用户问"XX余额"默认取期末余额（_end列），用户问"XX年初"取年初余额（_begin列）
23. balance_sheet_scope.gaap_type根据纳税人类型或用户明确指定确定：一般纳税人→ASBE，小规模纳税人→ASSE
24. 利润表视图: vw_profit_eas（企业会计准则）或 vw_profit_sas（小企业会计准则）
25. 利润表查询默认period_mode="month"，必须包含period_year和period_month
26. 利润表指标列使用英文物理字段名（如operating_revenue, operating_cost, total_profit, net_profit等）
27. 利润表time_range过滤: 用户问"本期"/"当月"→profit_time_range="本期"，用户问"本年累计"/"累计"→profit_time_range="本年累计"，默认"本年累计"

29. domain=cash_flow时，必须填写cash_flow_scope

32. 现金流量表指标列使用英文物理字段名（如operating_net_cash, investing_net_cash, financing_net_cash, net_increase_cash等）
33. 现金流量表time_range过滤: 用户问"本期"/"当月"→cash_flow_time_range="本期"，用户问"本年累计"/"累计"→cash_flow_time_range="本年累计"，默认"本年累计"
34. cash_flow_scope.accounting_standard根据纳税人类型确定：一般纳税人→企业会计准则，小规模纳税人→小企业会计准则
35. domain=cross_domain时，必须填写cross_domain_list（涉及的子域列表，如["profit","vat"]），并为每个子域填写对应的scope
36. 跨域查询示例：用户问"营业收入和增值税销售额"→domain=cross_domain, cross_domain_list=["profit","vat"]
37. 跨域检测验证规则：
   - 检查cross_domain_list中的每个域，确认用户问题中是否明确提到该域的特征词汇
   - 如果某个域仅因共享关键词被触发（如"应纳税额"），检查上下文是否真的属于该域
   - 示例：用户问"企业所得税应纳税额"，虽然包含"应纳税额"，但上下文是"企业所得税"，因此NOT包含VAT域
38. domain=financial_metrics时，必须填写financial_metrics_scope
38. 财务指标视图: vw_financial_metrics，指标列: metric_value, metric_unit, evaluation_level
39. 财务指标查询默认period_mode="month"，必须包含period_year和period_month
40. 财务指标通过metric_name或metric_code过滤具体指标，通过metric_category过滤指标类别
41. 财务指标metric_category有效值：盈利能力、偿债能力、营运能力、成长能力、现金流、税负率类、增值税重点指标、所得税重点指标、风险预警类
42. 用户问"全部指标"/"所有指标"时不加metric_name/metric_code过滤
43. domain=invoice时，必须填写invoice_scope
44. 发票视图: vw_inv_spec_purchase（进项/采购发票）或 vw_inv_spec_sales（销项/销售发票）
45. 发票方向判断：进项发票/采购发票→direction=purchase, views=["vw_inv_spec_purchase"]；销项发票/销售发票→direction=sales, views=["vw_inv_spec_sales"]；仅"发票"→direction=both, views=["vw_inv_spec_purchase","vw_inv_spec_sales"]
46. 发票查询默认period_mode="month"，必须包含period_year和period_month
47. 发票指标列: amount(金额), tax_amount(税额), total_amount(价税合计)
48. 发票维度列: invoice_pk, invoice_date, seller_name, buyer_name, invoice_type, invoice_status, is_positive, risk_level等
