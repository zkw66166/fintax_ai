# L2缓存跨域查询支持 - 快速参考

## 功能概述

L2缓存现在支持跨域查询，可以为跨域查询保存和复用SQL模板，显著提升性能。

## 工作原理

### 保存阶段
1. 用户发起跨域查询（如"今年各季度总资产、总负债和净利润情况"）
2. Pipeline生成多个子域SQL（balance_sheet, profit等）
3. 执行SQL并返回结果
4. **新增**: 将所有子域SQL模板化（替换 `:taxpayer_id` → `{{TAXPAYER_ID}}`）
5. **新增**: 保存为跨域L2模板文件

### 命中阶段
1. 用户发起相同查询（不同公司，但相同会计准则和纳税人类型）
2. L2缓存检测到匹配的跨域模板
3. **新增**: 实例化所有子域SQL（替换 `{{TAXPAYER_ID}}` → 当前公司ID）
4. **新增**: 执行所有子域SQL
5. **新增**: 合并结果（使用 `merge_cross_domain_results()`）
6. 返回结果（跳过LLM调用）

## 缓存键策略

跨域查询使用与单域查询相同的域感知缓存键策略：

- **财务报表子域** (balance_sheet, profit, cash_flow, account_balance):
  - 缓存键: `MD5(query | response_mode | fs | accounting_standard)`
  - 每个查询最多2个模板（企业会计准则, 小企业会计准则）

- **VAT子域**:
  - 缓存键: `MD5(query | response_mode | vat | taxpayer_type)`
  - 每个查询最多2个模板（一般纳税人, 小规模纳税人）

- **EIT子域**:
  - 缓存键: `MD5(query | response_mode | eit)`
  - 每个查询1个模板（无类型/准则区分）

- **跨域查询整体**:
  - 每个查询最多4个模板（2种会计准则 × 2种纳税人类型）
  - 逐步积累，不会一次性生成所有组合

## 模板文件结构

```json
{
  "cache_key": "abc123...",
  "query": "今年各季度总资产、总负债和净利润情况",
  "response_mode": "detailed",
  "cache_domain": "cross_domain",
  "subdomains": ["balance_sheet", "profit"],
  "taxpayer_type": "一般纳税人",
  "accounting_standard": "企业会计准则",
  "cross_domain_operation": "list",
  "sub_templates": [
    {
      "domain": "balance_sheet",
      "sql_template": "WITH latest AS (...) WHERE taxpayer_id = {{TAXPAYER_ID}} ..."
    },
    {
      "domain": "profit",
      "sql_template": "WITH latest AS (...) WHERE taxpayer_id = {{TAXPAYER_ID}} ..."
    }
  ],
  "intent": {
    "taxpayer_id": "91310115MA2KZZZZZZ",
    "period_year": 2026,
    ...
  },
  "created_at": 1709913000.0,
  "last_accessed": 1709913000.0,
  "hit_count": 0
}
```

## 性能对比

### 未命中缓存（首次查询）
1. LLM生成子域SQL: ~2-3秒
2. 执行SQL: ~0.5秒
3. 合并结果: ~0.1秒
4. **总耗时**: ~2.6-3.6秒

### 命中L2缓存
1. 实例化SQL模板: ~0.01秒
2. 执行SQL: ~0.5秒
3. 合并结果: ~0.1秒
4. **总耗时**: ~0.6秒

**性能提升**: ~77% (节省2-3秒LLM调用时间)

## 故障排查

### 问题1: 跨域查询未创建L2模板
**检查**:
- 查询是否成功（`result['success'] == True`）
- 是否有 `sub_results` 字段
- 每个子域SQL是否包含 `:taxpayer_id` 占位符

**日志关键字**: `[L2 Cache] Cross-domain templatize failed`

### 问题2: L2缓存未命中
**检查**:
- 查询文本是否完全一致（大小写、标点符号）
- 会计准则和纳税人类型是否匹配
- 响应模式是否一致（detailed/concise）

**日志关键字**: `[L2 Cache] Miss`

### 问题3: SQL执行失败
**检查**:
- 模板实例化是否正确（`{{TAXPAYER_ID}}` 是否被替换）
- 公司ID是否存在于数据库
- 数据库连接是否正常

**日志关键字**: `[L2 Cache] SQL failed`

## 配置选项

在 `config/settings.py` 中：

```python
# L2缓存总开关
QUERY_CACHE_ENABLED_L2 = True

# L2缓存最大文件数（包括单域和跨域）
QUERY_CACHE_MAX_FILES_L2 = 500

# L2缓存文件前缀
QUERY_CACHE_L2_PREFIX = "template_"
```
