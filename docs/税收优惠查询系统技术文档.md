## 二、核心查询流程

### 2.1 四层查询策略

| 优先级 | 策略 | 触发条件 | 示例 |
|--------|------|----------|------|
| **1** | 精准结构化查询 | 用户明确指定税种 | "企业所得税小微优惠" |
| **2** | 跨税种实体搜索 | 有实体关键词，无税种 | "小微企业优惠有哪些" |
| **3** | 关键词搜索 | 上述都无结果 | "关于软件的减免" |
| **4** | 原文兜底搜索 | 所有策略都失败 | (非常生僻的短语) |

### 2.2 策略1: 精准结构化查询

当用户**明确指定税种**时使用:

```python
# 伪代码逻辑
if tax_type and tax_type_source == "explicit":
    results = self.structured_search(tax_type, entity_keywords, limit)
    # SQL: WHERE tax_type = ? AND incentive_method LIKE ?
```

**示例查询**:
```sql
SELECT * FROM tax_incentives 
WHERE tax_type = '企业所得税' 
  AND (incentive_method LIKE '%减征%' OR incentive_method LIKE '%免税%')
LIMIT 20
```

### 2.3 策略2: 跨税种实体搜索

当问题包含**实体关键词**（如"小微企业"、"高新技术"）但未指定税种:

```python
# 自动跨所有税种搜索
results = self.entity_search(entity_keywords, limit)
# 搜索字段: incentive_items, detailed_rules, qualification, incentive_method
```

### 2.4 策略3: 关键词搜索

```python
# 提取关键词后，使用 LIKE 进行多字段搜索
keyword_search(keywords, limit=50)

# 搜索字段:
# - tax_type
# - incentive_items  
# - qualification
# - detailed_rules
# - keywords
# - explanation
# - incentive_method
# - legal_basis
```

### 3.3 热更新机制

```python
def _load_config(self):
    """每次调用检查文件修改时间,自动热更新"""
    current_mtime = os.path.getmtime(self.config_path)
    if current_mtime != self._config_mtime:
        # 文件已修改,重新加载
        with open(self.config_path, 'r', encoding='utf-8') as f:
            self._config = json.load(f)
```

**使用方式**: 修改 `config/tax_query_config.json` 后，无需重启服务，配置自动生效。

## 六、LLM 增强机制

### 6.1 税种推理

当用户**未明确指定税种**时，使用 DeepSeek 推理:

```python
def _infer_tax_type_with_llm(self, question: str):
    prompt = """请根据以下问题判断涉及的税种。
    
    问题: {question}
    
    税种列表: 增值税, 企业所得税, 个人所得税...
    
    判断规则:
    1. 如果涉及"税前扣除"、"加计扣除"、"研发费用"→企业所得税
    2. 如果涉及"专项附加扣除"→个人所得税
    3. 如果涉及"进项税"、"销项税"、"抵扣"→增值税
    
    请只返回税种名称或"无法判断"。"""
```

### 6.2 优惠项目关键词提取

```python
def _extract_project_keywords_with_llm(self, question: str):
    prompt = """请从以下问题中提取税收优惠相关的项目关键词。
    
    问题: {question}
    
    项目类型: 产品(粮食/软件), 服务(会议展览), 
              行业(出版/科研), 企业类型(高新/小微)...
    
    要求: 只提取2-6个字的关键词,最多3个"""
```

## 九、错误处理

```python
try:
    results = db_query.search(question, limit=20)
except Exception as e:
    yield send_content(f"⚠️ 查询出错: {str(e)}\n")
```

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 搜索无结果 | 关键词不在数据库 | 尝试更通用的关键词 |
| 结果不准确 | 同义词未配置 | 联系管理员添加同义词 |
| 响应慢 | 数据量大 | 考虑添加 FTS 索引 |
