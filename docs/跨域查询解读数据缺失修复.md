# 跨域查询解读数据缺失修复

**日期**: 2026-03-09
**问题**: 跨域查询返回正确结果，但 LLM 解读错误声称数据缺失
**影响范围**: 跨域 list 操作（如"去年第四季度总资产、净利润、应纳企业所得税额情况"）

---

## 问题描述

### 用户报告
查询"去年第四季度总资产、净利润、应纳企业所得税额情况"时：
- ✅ 后端正确返回三个子域的数据（balance_sheet, profit, eit）
- ✅ 前端正确显示所有三个指标
- ❌ LLM 解读错误声称"净利润"和"应纳企业所得税额"数据缺失

### 根本原因

**跨域 list 操作的数据结构**:
```json
{
  "display_data": {
    "table": {"headers": [], "rows": []},  // 主表为空
    "sub_tables": [                         // 实际数据在这里
      {
        "domain": "balance_sheet",
        "domain_cn": "资产负债表",
        "table": {"headers": [...], "rows": [...]}
      },
      {
        "domain": "profit",
        "domain_cn": "利润表",
        "table": {"headers": [...], "rows": [...]}
      },
      {
        "domain": "eit",
        "domain_cn": "企业所得税",
        "table": {"headers": [...], "rows": [...]}
      }
    ]
  }
}
```

**问题所在**:
- `modules/interpretation_service.py` 的 `_format_data_for_prompt()` 函数只读取 `display_data.table`
- 对于跨域 list 操作，`display_data.table` 是空的（设计如此）
- 实际数据在 `display_data.sub_tables` 中，但解读服务没有提取
- 导致 LLM 看到空数据，错误报告数据缺失

---

## 修复方案

### 1. 增强数据提取逻辑

**文件**: `modules/interpretation_service.py`
**函数**: `_format_data_for_prompt()` (line 167-242)

**修改内容**:
```python
# 跨域 list 操作：从 sub_tables 提取数据
sub_tables = dd.get('sub_tables', [])
if sub_tables:
    # 合并所有子域的表格数据
    all_headers_set = set()
    all_rows = []

    for st in sub_tables:
        domain_cn = st.get('domain_cn', st.get('domain', ''))
        st_table = st.get('table', {})
        st_headers = st_table.get('headers', [])
        st_rows = st_table.get('rows', [])

        # 为每行添加域标识
        for row in st_rows:
            row_with_domain = {'数据来源': domain_cn}
            row_with_domain.update(row)
            all_rows.append(row_with_domain)

        all_headers_set.update(st_headers)

    # 构建统一表头（数据来源 + 所有指标列）
    headers = ['数据来源'] + sorted([h for h in all_headers_set if h != '数据来源'])

    # 格式化为文本表格
    display_rows = all_rows[:50]
    lines = [" | ".join(headers)]
    for row in display_rows:
        lines.append(" | ".join(str(row.get(h, '')) for h in headers))
    if len(all_rows) > 50:
        lines.append(f"... (共{len(all_rows)}行，仅展示前50行)")
    parts.append(f"查询结果：\n{chr(10).join(lines)}")

    # 添加子域数据摘要
    summary_lines = ["各子域数据概览："]
    for st in sub_tables:
        domain_cn = st.get('domain_cn', '')
        st_table = st.get('table', {})
        row_count = len(st_table.get('rows', []))
        col_count = len(st_table.get('headers', []))
        summary_lines.append(f"  - {domain_cn}: {row_count}行 × {col_count}列")
    parts.append("\n".join(summary_lines))

# 单域或其他跨域操作
else:
    # 原有逻辑保持不变
    ...
```

**关键改进**:
1. 检测 `sub_tables` 存在时，从中提取所有子域数据
2. 为每行添加"数据来源"列，标识数据来自哪个域
3. 合并所有子域的表头和数据行
4. 添加子域数据摘要（行数×列数），帮助 LLM 理解数据结构
5. 使用 `elif` 确保只走一条数据提取路径

### 2. 更新场景检测

**文件**: `modules/interpretation_service.py`
**函数**: `detect_scenario()` (line 65-101)

**修改内容**:
```python
def detect_scenario(result: dict) -> dict:
    """根据查询结果分类场景"""
    # 优先判断特殊路径
    if result.get('metric_results'):
        return {'scenario': 'metric_computed'}

    # 检查跨域查询：sub_results 或 sub_tables
    if result.get('cross_domain_summary') or result.get('sub_results'):
        return {'scenario': 'cross_domain'}

    dd = result.get('display_data', {})
    if dd.get('sub_tables'):  # 新增：检测 sub_tables
        return {'scenario': 'cross_domain'}

    # 其他场景检测逻辑保持不变
    ...
```

**关键改进**:
- 新增 `sub_tables` 检测，确保跨域 list 操作被正确识别为 `cross_domain` 场景

### 3. 增强跨域场景指令

**文件**: `modules/interpretation_service.py`
**常量**: `SCENARIO_INSTRUCTIONS['cross_domain']` (line 141-149)

**修改内容**:
```python
'cross_domain': (
    "本次查询涉及跨域数据（如利润与增值税、资产负债表与利润表等）。请分析：\n"
    "- **数据完整性检查**：首先确认各域数据是否完整返回（检查\"数据来源\"列）\n"
    "- **各域数据解读**：分别解读各域的关键指标和数值水平\n"
    "- **跨域关联分析**：分析各域数据之间的逻辑一致性和关联关系\n"
    "- **数值匹配度**：评估跨域指标的数值匹配度（如净利润与所得税、营业收入与销项税额）\n"
    "- **异常识别**：如存在背离或差异，分析可能原因\n"
    "- **注意**：只有当数据确实缺失时才报告缺失，不要因为数据来自不同域就误判为缺失"
),
```

**关键改进**:
1. 新增"数据完整性检查"指令，要求 LLM 首先检查"数据来源"列
2. 新增"各域数据解读"指令，要求分别解读各域数据
3. 新增"注意"警告，明确指出不要误判数据缺失
4. 保持原有的跨域关联分析要求
