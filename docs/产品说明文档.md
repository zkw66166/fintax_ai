## 1. 产品概述

### 1.1 产品定位

FinTax AI 是一款面向中国企业的**智能税务与财务咨询平台**，通过自然语言处理技术，将复杂的财税数据查询转化为简单的对话交互。系统采用两阶段 NL2SQL（自然语言转SQL）技术，结合大语言模型（LLM）的语义理解能力和结构化数据库的精确查询能力，为企业提供实时、准确的财税数据分析服务。

### 1.2 核心价值主张

| 价值维度 | 具体描述 |
|---------|---------|
| **降低专业门槛** | 非财税专业人员也能快速获取财税数据，无需学习复杂的报表结构和税法条款 |
| **提升决策效率** | 秒级响应查询，支持多轮对话快速获得所需信息 |
| **数据准确可靠** | 基于结构化SQLite数据库存储，确保查询结果准确无误，避免大模型幻觉 |
| **全域数据整合** | 整合增值税、企业所得税、财务报表、发票、财务指标等9大数据域 |
| **智能税务筹划** | 结合1522条税收优惠政策库和外部法规知识库，提供综合税务建议 |

### 1.3 技术特点

- **四路由智能架构**：财务数据查询、税收优惠检索、法规知识库、混合分析四大路由智能分流
- **两阶段NL2SQL**：Stage 1 意图解析 + Stage 2 SQL生成，确保查询准确性
- **确定性计算路径**：财务指标和概念查询无需LLM参与，直接SQL+Python计算
- **多轮对话支持**：支持上下文继承和跨路由综合分析，实现连续对话体验
- **双层缓存机制**：L1结果缓存 + L2模板缓存，大幅提升响应速度
- **SSE流式输出**：实时流式响应，提升用户体验

### 1.4 目标用户

| 用户类型 | 核心需求 | 典型使用场景 |
|---------|---------|-------------|
| **企业财务人员** | 日常财税数据查询、报表分析、税负计算 | "华兴科技2025年1月增值税销项税额是多少？" |
| **税务顾问/事务所** | 多客户管理、税务筹划方案、合规检查 | 对比分析多家客户的税负情况 |
| **集团企业财务总监** | 多子公司数据对比、集团财务分析 | 汇总分析旗下各子公司的财务状况 |
| **企业管理层** | 经营决策支持、财务健康度监控 | 查看企业健康评分和关键财务指标 |
| **系统管理员** | 用户管理、权限配置、系统监控 | 管理系统用户和数据访问权限 |

### 1.5 数据覆盖范围

| 维度 | 覆盖范围 |
|-----|---------|
| **企业数量** | 6家样本企业 |
| **数据域** | 9大核心财税数据域 |
| **时间跨度** | 2023年1月 - 2026年2月 |
| **会计准则** | 企业会计准则(ASBE)、小企业会计准则(ASSE) |
| **纳税人类型** | 一般纳税人、小规模纳税人 |
| **政策数据** | 1522条税收优惠政策 |

#### 3.2.2 路由决策层级

| 层级 | 名称 | 决策逻辑 | 优先级 |
|-----|------|---------|--------|
| Layer -2 | 财务数据优先 | 数据类关键词 + 税种/财务术语同时出现 | 最高 |
| Layer -1 | 知识库优先 | 流程/操作类关键词（指南、申报流程等） | 次高 |
| Layer 0 | 企业数据查询 | 纳税人名称模糊匹配 或 时间+金额模式 | 中 |
| Layer 1 | 税收优惠 | 优惠关键词（减免、抵扣、退税等） | 低 |
| Default | 法规知识 | 无匹配则路由到法规知识库 | 最低 |

#### 3.2.3 混合分析触发条件

当以下**全部条件**满足时，触发混合分析路由：

1. `MIXED_ANALYSIS_ENABLED = True`（配置开关）
2. 用户启用多轮对话（`conversation_depth ≥ 2`）
3. 历史记录非空
4. 历史包含 **≥2种不同路由**（如tax_incentive + financial_data）
5. LLM检测到综合分析需求（关键词：综合、匹配、建议、筹划、优化）

- 秒级响应，自动关联数据
- 图表可视化，趋势一目了然
- 支持多轮对话，连续追问

#### 场景2：税务顾问多客户管理

**痛点**：
- 管理多家客户，数据切换繁琐
- 需要对比不同客户的财税状况
- 税务筹划需要综合多方面信息

**FinTax AI解决方案**：
- 客户组合视图，一览所有客户健康状况
- 快速切换公司，无需重新登录
- 混合分析引擎，自动匹配政策与数据
- 1522条政策库，智能检索

#### 场景3：集团企业多子公司对比

**痛点**：
- 子公司数据分散在不同系统
- 跨公司对比需要手动整理
- 难以发现异常和风险
- 缺乏统一的数据质量标准

**FinTax AI解决方案**：
- 多企业数据对比功能
- 统一数据质量检查标准
- 批量质量检查，快速发现问题
- 健康评分排名，识别风险企业

#### 场景4：企业管理层决策支持

**痛点**：
- 财务报表专业性强，难以理解

## 12. 常见问题

### 12.1 查询问题排查

#### Q1: 查询结果为空

**可能原因**：
1. 查询期间没有数据（样本数据覆盖2023.01-2026.02）
2. 公司名称识别错误
3. 域检测错误

**解决方案**：
- 使用完整公司名称
- 检查查询期间是否在数据覆盖范围内
- 添加明确的域关键词（增值税、利润表等）

#### Q2: SQL审计失败

**可能原因**：
1. 缺少必需过滤条件（taxpayer_id、period）
2. 查询了不允许的视图
3. 缺少LIMIT子句

**解决方案**：
- LLM会自动重试一次
- 如仍失败，使用深度模式或调整查询

#### Q3: 多轮对话不生效

**可能原因**：
1. 未启用多轮对话开关
2. 不在beta用户白名单
3. 历史记录未正确传递

**解决方案**：
- 前端勾选"多轮对话"
- 联系管理员添加到白名单
- 刷新页面重试

### 12.2 性能问题排查

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| 响应慢 | 使用深度模式 | 切换到快速/思考模式 |
| 响应慢 | 复杂跨域查询 | 拆分为多个简单查询 |
| 响应慢 | LLM服务延迟 | 等待或稍后重试 |
| 响应慢 | 缓存未命中 | 第二次查询会更快 |

### 12.3 权限问题排查

| 错误码 | 含义 | 解决方案 |
|-------|------|---------|
| 401 | 未登录或token过期 | 重新登录 |
| 403 | 无公司访问权限 | 联系管理员分配权限 |
| 403 | 角色权限不足 | 联系管理员提升权限 |
