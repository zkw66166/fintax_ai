# 跨域查询性能优化方案

## Context

跨域查询（cross_domain）性能瓶颈分析：
- 单域查询：2次LLM调用（Stage1意图 + Stage2 SQL），约4-6秒
- 3域跨域查询：1次Stage1 + 3次Stage2 = 4次LLM调用，**全部串行执行**，约10-18秒
- 每次LLM调用创建新的OpenAI客户端，浪费连接池
- 跨域合并结果未缓存，重复查询需重新执行所有子域

核心瓶颈：`mvp_pipeline.py:382-478` 的 `for sd in cross_list` 循环中，每个子域的 `generate_sql()` LLM调用串行等待。

## 优化方案（3项，按优先级排序）

### 优化1：跨域子查询并行执行（预计提速2-3x）

**文件**: `mvp_pipeline.py`

将 `_run_cross_domain_pipeline()` 中的子域循环改为并行执行：

```
当前（串行）：
  for sd in cross_list:
      build_sub_intent → inject_constraints → generate_sql(LLM) → audit → execute

改为（并行）：
  Phase 1: 串行构建所有 sub_intent + constraints（快，无LLM）
  Phase 2: ThreadPoolExecutor 并行执行 generate_sql + audit + retry（LLM瓶颈）
  Phase 3: 串行执行所有 SQL 查询（快，<100ms/条）
```

实现要点：
- 使用 `concurrent.futures.ThreadPoolExecutor(max_workers=len(cross_list))`
- 提取子域处理逻辑为独立函数 `_process_subdomain(sd, sub_constraints, domain)`
- 该函数只做 LLM调用 + 审核 + 重试，返回 `(domain, sql)` 或 `(domain, error)`
- SQL执行留在主线程（避免SQLite多线程问题）
- Stage2 SQL缓存在并行线程中仍然生效（LRUCache加锁保护）

### 优化2：复用OpenAI客户端（减少连接开销）

**文件**: `modules/sql_writer.py`, `modules/intent_parser.py`

当前每次调用 `generate_sql()` 和 `parse_intent()` 都创建新的 `OpenAI()` 客户端。改为模块级单例：

```python
# modules/sql_writer.py
_client = None
def _get_client():
    global _client
    if _client is None:
        _client = OpenAI(api_key=LLM_API_KEY, base_url=LLM_API_BASE, timeout=LLM_TIMEOUT)
    return _client
```

同样修改 `intent_parser.py`。httpx底层连接池可复用TCP连接，减少TLS握手开销。

### 优化3：缓存层增强

**文件**: `modules/cache_manager.py`, `mvp_pipeline.py`

#### 3a. SQL执行结果缓存
缓存 `(sql, params)` → `rows` 的映射，避免重复DB查询：
- 新增 `_result_cache: LRUCache`（max_size=200, ttl=1800s）
- cache_key = `hash(sql + sorted(params))`
- 在 `conn.execute(sql, params)` 前后加缓存读写

#### 3b. 跨域合并结果缓存
缓存整个跨域查询的最终结果：
- cache_key = `hash(normalized_query + taxpayer_id + period + cross_domain_list)`
- 在 `_run_cross_domain_pipeline()` 入口检查缓存，出口写入缓存
- TTL=1800s（与Stage1一致）

#### 3c. LRUCache线程安全
为并行执行加 `threading.Lock()`：
- `get()` 和 `set()` 方法加锁
- 保证并行LLM调用时缓存读写安全

| 文件 | 修改内容 |

## 验证方式

1. 运行 `python test_performance.py` 对比优化前后耗时
2. 手动测试跨域查询：
   ```
   华兴科技2023年到2025年营业收入、销项税和经营现金流净额
   ```
   观察控制台输出中各子域是否并行执行（时间戳接近）
3. 重复执行同一跨域查询，验证缓存命中
4. 运行 `python test_comprehensive.py` 确保功能不退化
