# 优化前端查询结果展示 — 实施计划

## Context

fintax_ai 的 `app.py` 中 `format_results_table()` 直接将管线返回的 `list[dict]` 渲染为 Markdown 表格，存在以下问题：
1. 列名全英文（`output_tax`, `net_profit`），中文用户无法阅读
2. 大金额无格式化（`150000000.0` 原样显示）
3. 内部元数据列暴露（`revision_no`, `etl_batch_id`, `etl_confidence` 等）
4. 所有域统一处理，无差异化展示（单行/多行、标准/跨域/指标）
5. None 值显示为空字符串
6. 计算指标的 `sources` 显示为原始 JSON

数据库中已有 `*_column_mapping` 表和 `*_item_dict` 表，提供 column_name → business_name 的中文映射，但当前未被展示层使用。

**约束**：9 域查询管线逻辑不变，`mvp_pipeline.py` 返回数据结构不变，仅改展示层。

---

## 实施步骤

### Step 1: 新建 `modules/display_formatter.py`

核心模块，约 300 行，包含以下组件：

#### 1a. 常量定义
- `HIDDEN_COLUMNS`: 需隐藏的内部列集合（`revision_no`, `etl_batch_id`, `etl_confidence`, `submitted_at`, `source_doc_id`, `source_unit`, `filing_id`）
- `COMMON_COLUMN_CN`: 通用元数据列的中英映射（`taxpayer_id`→`纳税人识别号`, `period_year`→`年度` 等，约 50 项）
- `PERCENTAGE_COLUMNS`: 百分比列集合 + 识别规则（列名含 `_rate`/`_ratio`/`(%)` 后缀）
- `DOMAIN_CN_DISPLAY`: 域名中文显示名（`vat`→`增值税` 等）

#### 1b. `ColumnMapper` 类（懒加载单例）
- 首次调用 `translate()` 时从数据库加载所有映射
- 加载源：
  - `vat_general_column_mapping` / `vat_small_column_mapping`（column_name → business_name）
  - `eit_annual_main_column_mapping` / `eit_quarter_main_column_mapping`
  - `fs_balance_sheet_item_dict`（item_code → `{item_name}(年初余额)` / `{item_name}(期末余额)`）
  - `fs_income_statement_item_dict`（item_code → item_name）
  - `fs_cash_flow_item_dict`（item_code → item_name）
- `translate(column_name, domain, view)` 查找优先级：
  1. 精确 view 映射（如 `vw_profit_eas`）
  2. domain → 候选 views 映射（通过 `schema_catalog.DOMAIN_VIEWS`）
  3. `COMMON_COLUMN_CN` 通用映射
  4. 跨域前缀拆分（`profit_net_profit` → `利润表-净利润`）
  5. 原始列名兜底

#### 1c. 数值格式化函数
- `format_number(value, is_percentage)`:
  - None → `"-"`
  - 非数值 → `str(value)`
  - 百分比 → `"25.50%"`
  - ≥1亿 → `"1.50亿"`
  - ≥1万 → `"12.35万"`
  - 其他 → 千分位 + 2位小数

#### 1d. 视图推断函数
- `_infer_domain_and_view(result)`: 从 `result['intent']` 提取 domain 和 view
  - intent 中 `{domain}_scope.views[0]` 即为精确视图名（如 `vw_profit_eas`）
  - 兜底用 `get_scope_view()` 推断

#### 1e. 格式化函数
- `_format_as_kv_list()`: 单行少列 → 列表格式（`- **指标名**: 值`）
- `_format_as_table()`: 多行 → Markdown 表格（中文表头 + 格式化数值）
- `_format_metric_computed()`: 计算指标专用（`**资产负债率**: 45.67%`，sources 展开为可读文本）
- `_format_cross_domain()`: 跨域结果（摘要 + 表格，保留已有中文计算列）
- `format_display(result)`: 主入口，根据结果类型分发

### Step 2: 修改 `app.py`

仅改 `_format_result()` 函数中 3 行（第 114-116 行）：

```python
# Before:
elif result.get('success'):
    table = format_results_table(result['results'])
    main_output = f"✅ 查询成功（{len(result['results'])} 行）\n\n{table}"

# After:
elif result.get('success'):
    from modules.display_formatter import format_display
    formatted = format_display(result)
    row_count = len(result.get('results', []))
    main_output = f"✅ 查询成功（{row_count} 行）\n\n{formatted}"
```

保留原 `format_results_table()` 函数不删除（可用于调试）。

不改动：流式逻辑、Gradio 布局、历史持久化、管线模块。

### Step 3: 新建单元测试 `tests/test_display_formatter.py`

测试覆盖：
- `format_number()`: None、亿、万、小数、负数、百分比、零
- `ColumnMapper.translate()`: 通用列、VAT 列、BS 列、未知列兜底
- `format_display()`: 标准单行（KV 列表）、标准多行（表格）、计算指标、跨域结果
- 隐藏列验证：输出中不含 `revision_no` 等

### Step 4: 集成测试验证

1. `python -m unittest tests.test_display_formatter` — 单元测试
2. `python test_comprehensive.py` — 57 题 9 域全量回归（验证管线不受影响）
3. 手动验证代表性查询的展示效果

## 与参考文档的冲突核对

| # | 参考文档方案 | fintax_ai 采用方案 | 理由 |
|---|-------------|-------------------|------|
| 1 | React + Chart.js 前端 | Gradio Markdown | 技术栈不同，不引入前端框架 |
| 2 | SSE 多事件类型 | 保持现有 stage/chunk/done | 已满足需求 |
| 3 | 统一 metric_name/value 结构 | 各域保持原有 dict 结构 | 展示层适配，不改管线 |
| 4 | 后端 format_results() 方法 | `display_formatter.py` | 一致思路，独立模块 |
| 5 | calculate_comparison() | 跨域已有 compare/ratio/reconcile | 在展示层增强箭头标记 |

无逻辑冲突。参考文档的数值格式化（亿/万转换）和列表/表格策略直接借鉴。

## 验证方法

1. 单元测试：`python -m unittest tests.test_display_formatter`
2. 回归测试：`python test_comprehensive.py`（期望 ~96% 通过率不变）
3. 手动验证 6 个代表性查询：
   - VAT 单值：`查询华兴科技2025年1月的销项税额` → KV 列表 + 中文列名
   - VAT 趋势：`华兴科技2025年1月到3月的应纳税额趋势` → 表格 + 格式化金额
   - 资产负债表：`华兴科技2025年1月资产负债表的货币资金` → `货币资金(期末余额)` 而非 `cash_end`
   - 计算指标：`华兴科技2025年3月的资产负债率` → `**资产负债率**: 45.67%`
   - 跨域对比：`华兴科技2025年3月利润表营业收入和增值税销项税额对比` → 中文表头 + 差异列
   - 税收优惠：`高新技术企业有哪些企业所得税优惠` → 不变（文本路由）
