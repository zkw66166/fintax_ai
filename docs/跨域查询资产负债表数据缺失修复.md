# Cross-Domain Query Missing Balance Sheet Data Fix

**Date**: 2026-03-09
**Status**: ✅ Fixed and Verified

## Problem

Cross-domain query "今年各季度总资产、总负债和净利润情况" only returned profit data (净利润), missing balance sheet data (总资产、总负债).

### Evidence from Logs

```
[跨域] 原始metrics: ['total_assets_end', 'total_liabilities_end', 'net_profit']
[balance_sheet] 过滤后metrics: []
[balance_sheet] 未匹配的metrics: ['total_assets_end', 'total_liabilities_end', 'net_profit']
[profit] 过滤后metrics: ['net_profit']

[balance_sheet] 返回 0 行  ❌
[profit] 返回 2 行  ✅
```

## Root Cause

**Metric Name Mismatch**: Stage 1 LLM extracted incorrect metric names for balance sheet items.

1. **User query**: "总资产" (total assets), "总负债" (total liabilities)
2. **Stage 1 extracted**: `total_assets_end`, `total_liabilities_end` ❌
3. **Actual column names**: `assets_end`, `liabilities_end` ✅
4. **Result**: Metrics didn't match → filtered out → 0 rows

### Why This Happened

The Stage 1 prompt (`prompts/stage1_system.txt` lines 62-81) had metric mapping examples for EIT and profit domains, but **NO examples for balance sheet items**. The LLM guessed `total_assets_end` instead of the correct `assets_end`.

## Solution

Added comprehensive metric mappings for ALL 7 domains to Stage 1 prompt (lines 62-221).

### File Modified

- `prompts/stage1_system.txt` (lines 62-81 replaced with 160 lines)
