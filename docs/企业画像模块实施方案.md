# 企业画像模块实施方案

> 基于 mcp_coze 企业画像参考设计 + fintax_ai 现有架构，从 fintax_ai.db 取数实现

## 一、需求分析与设计决策

### 1.1 数据源可用性分析

根据「企业画像指标与数据源对应表」，14 大画像模块中，fintax_ai.db 可直接支撑的模块：

| 模块 | 可用度 | 数据来源 |
|------|--------|----------|
| ① 企业身份画像 | ★★★★ | taxpayer_info（缺经营范围/注册地址/注册资本，需扩展字段） |
| ② 股权与治理画像 | ★☆☆☆ | 缺失（需新建 company_shareholders 表，EIT 年度有股东数据） |
| ③ 组织与人力画像 | ★☆☆☆ | 缺失（需新建 employee_structure 表，EIT 季度有 employee_avg） |
| ④ 财务画像 | ★★★★★ | 资产负债表 + 利润表 + 现金流量表 + financial_metrics |
| ⑤ 业务运营画像 | ★★★☆ | 利润表(营收) + 发票(数量统计) |
| ⑥ 研发创新画像 | ★★☆☆ | 利润表(研发费用) + EIT(研发加计扣除) |
| ⑦ 税务画像 | ★★★★★ | taxpayer_info + VAT申报 + EIT申报 + financial_metrics |
| ⑧ 跨境业务画像 | ★★☆☆ | EIT年度(境外所得) + VAT(出口销售) |
| ⑨ 合规风险画像 | ★★☆☆ | financial_metrics(风险指标) + user_query_log(申报记录) |
| ⑩ 外部关系画像 | ☆☆☆☆ | 完全缺失 |
| ⑪ 数字化画像 | ☆☆☆☆ | 完全缺失 |
| ⑫ ESG画像 | ★☆☆☆ | 利润表(捐赠支出) |
| ⑬ 政策匹配画像 | ★★☆☆ | EIT(优惠明细) + VAT(即征即退) |
| ⑭ 特殊业务画像 | ★☆☆☆ | 发票(软件收入) + VAT(退税) |

### 1.2 MVP 范围决策

**第一期实现（本次）**：聚焦数据充足的 7 个核心模块
1. ① 企业身份画像 — 基本信息展示
2. ④ 财务画像 — 核心模块，9 个子维度全部实现
3. ⑤ 业务运营画像 — 营收 + 发票统计
4. ⑥ 研发创新画像 — 研发费用 + 研发强度
5. ⑦ 税务画像 — 税负分析
6. ⑧ 跨境业务画像 — 境外所得 + 出口
7. ⑨ 合规风险画像 — 财务风险指标

**第二期（后续）**：需补充数据源的模块
- ② 股权与治理、③ 组织与人力、⑩ 外部关系、⑪ 数字化、⑫ ESG、⑬ 政策匹配、⑭ 特殊业务

缺失数据的模块在前端显示为「暂无数据」占位卡片，保留完整 14 模块框架。

#### Step 1.2：创建 profile_service.py（核心数据聚合服务）

新建 `modules/profile_service.py`，实现 `get_company_profile(taxpayer_id, year)` 函数。

该函数执行 21 个子查询，返回结构化 JSON。每个子查询对应一个私有函数：

```python
# modules/profile_service.py

def get_company_profile(taxpayer_id: str, year: int) -> dict:
    """聚合企业画像全部数据，返回结构化 dict"""
    conn = sqlite3.connect(str(DB_PATH))
    conn.row_factory = sqlite3.Row
    try:
        result = {
            "basic_info": _query_basic_info(conn, taxpayer_id),
            "financial_summary": _query_financial_summary(conn, taxpayer_id, year),
            "financial_metrics": _query_financial_metrics(conn, taxpayer_id, year),
            "asset_structure": _query_asset_structure(conn, taxpayer_id, year),
            "profit_data": _query_profit_data(conn, taxpayer_id, year),
            "cash_flow": _query_cash_flow(conn, taxpayer_id, year),
            "growth_metrics": _query_growth_metrics(conn, taxpayer_id, year),
            "tax_summary": _query_tax_summary(conn, taxpayer_id, year),
            "invoice_summary": _query_invoice_summary(conn, taxpayer_id, year),
            "rd_innovation": _query_rd_innovation(conn, taxpayer_id, year),
            "cross_border": _query_cross_border(conn, taxpayer_id, year),
            "compliance_risk": _query_compliance_risk(conn, taxpayer_id, year),
            # 以下为第二期，暂返回空
            "shareholders": None,
            "employee_structure": None,
            "external_relations": None,
            "digitalization": None,
            "esg": None,
            "policy_matching": None,
            "special_business": None,
        }
        return result
    finally:
        conn.close()
```

各子查询函数的 SQL 数据源映射：

| 子查询函数 | 数据源视图/表 | 核心 SQL 逻辑 |
|-----------|-------------|--------------|
| `_query_basic_info` | `taxpayer_info` | 直接 SELECT 全字段 |
| `_query_financial_summary` | `vw_balance_sheet_eas/sas` | 取年末月(12月)资产总额、负债总额、所有者权益 |
| `_query_financial_metrics` | `financial_metrics_item` | WHERE taxpayer_id AND period_year，按 metric_category 分组 |
| `_query_asset_structure` | `vw_balance_sheet_eas/sas` | 取流动资产、固定资产、无形资产、其他资产的期末值 |
| `_query_profit_data` | `vw_profit_eas/sas` | 取12月本年累计：营业收入、营业成本、净利润、各费用 |
| `_query_cash_flow` | `vw_cash_flow_eas/sas` | 取12月本年累计：经营/投资/筹资活动现金流净额 |
| `_query_growth_metrics` | `vw_profit_eas/sas` + `vw_balance_sheet_eas/sas` | 对比 year 与 year-1 的营收、净利润、资产总额，计算增长率 |
| `_query_tax_summary` | `vw_vat_return_general/small` + `vw_eit_annual_main` | VAT: SUM(total_tax_payable) 全年; EIT: actual_tax_payable |
| `_query_invoice_summary` | `inv_spec_purchase` + `inv_spec_sales` | COUNT(*) + SUM(total_amount) 按年度 |
| `_query_rd_innovation` | `vw_profit_eas/sas` | 取 rd_expense (研发费用)，计算研发强度 = rd_expense / revenue |
| `_query_cross_border` | `vw_eit_annual_main` | less_foreign_income, add_foreign_tax_due, less_foreign_tax_credit_amount |
| `_query_compliance_risk` | `financial_metrics_item` | 筛选 metric_category = '风险预警类' |

关键实现细节：
- 根据 `taxpayer_info.accounting_standard` 自动路由到 EAS/SAS 视图
- 年度数据取 `period_month = 12` 的本年累计（`time_range = '本年累计'`）
- 增长率计算：`(current - previous) / abs(previous) * 100`，previous 为 0 时标记为 null
- 所有金额统一为「元」，前端负责万元/亿元格式化

#### Step 3.2：财务画像模块 (FinancialModule) — 核心

最复杂的模块，分 9 个子区域。布局采用 CSS Grid 2×5 排列。

**子区域 A：财务综合能力**
- 综合评分（来自 financial_metrics 综合计算）
- 5 维能力进度条：盈利能力、偿债能力、运营效率、成长能力、现金流
- 数据源：`financial_metrics_item` 按 category 分组取平均评分

**子区域 B：资产结构分析**
- 资产总额 + 同比增长率
- 饼图：流动资产 / 固定资产 / 无形资产 / 其他
- 数据源：`vw_balance_sheet_eas/sas` 取 ASSETS_end, CURRENT_ASSETS_end, FIXED_ASSETS_end, INTANGIBLE_ASSETS_end

**子区域 C：负债与权益**
- 资产负债率 + EvalLabel
- 柱状图：负债 vs 所有者权益
- 数据源：LIABILITIES_end, EQUITY_end

**子区域 D：盈利能力**
- 营业收入 + 同比增长率
- 毛利率、净利润、净利率
- 数据源：`vw_profit_eas/sas` (operating_revenue, operating_cost, net_profit)
- 毛利率 = (revenue - cost) / revenue

**子区域 E：偿债能力**
- 资产负债率、流动比率、速动比率
- 数据源：`financial_metrics_item` (debt_ratio, current_ratio, quick_ratio)

**子区域 F：运营效率**
- 应收款周转天数、应收款周转率、总资产周转率
- 数据源：`financial_metrics_item` (ar_turnover, asset_turnover)
- 周转天数 = 365 / 周转率

**子区域 G：成长能力**
- 资产增长率、营收增长率、净利润增长率
- 数据源：growth_metrics（后端已计算）

**子区域 H：成本费用结构**
- 销售费用、管理费用、研发费用、财务费用
- 费用率 = 费用 / 营业收入
- 数据源：`vw_profit_eas/sas`

**子区域 I：现金流量**
- 经营/投资/筹资活动现金流净额
- 数据源：`vw_cash_flow_eas/sas`

#### Step 5.2：MiniChart 组件

```jsx
export default function MiniChart({ type, data, height = 200 }) {
  // type: 'pie' | 'bar' | 'line'
  const ChartComponent = { pie: Pie, bar: Bar, line: Line }[type]
  return (
    <div style={{ height }}>
      <ChartComponent data={data} options={MINI_OPTIONS[type]} />
    </div>
  )
}
```

## 五、关键 SQL 查询参考

### 5.1 资产结构查询（企业会计准则）

```sql
SELECT
  MAX(CASE WHEN item_code='ASSETS' THEN ending_balance END) AS total_assets,
  MAX(CASE WHEN item_code='CURRENT_ASSETS' THEN ending_balance END) AS current_assets,
  MAX(CASE WHEN item_code='FIXED_ASSETS' THEN ending_balance END) AS fixed_assets,
  MAX(CASE WHEN item_code='INTANGIBLE_ASSETS' THEN ending_balance END) AS intangible_assets,
  MAX(CASE WHEN item_code='LIABILITIES' THEN ending_balance END) AS total_liabilities,
  MAX(CASE WHEN item_code='EQUITY' THEN ending_balance END) AS total_equity
FROM fs_balance_sheet_item
WHERE taxpayer_id = ?
  AND period_year = ?
  AND period_month = 12
  AND gaap_type = 'ASBE'
  AND revision_no = (
    SELECT MAX(revision_no) FROM fs_balance_sheet_item
    WHERE taxpayer_id = ? AND period_year = ? AND period_month = 12 AND gaap_type = 'ASBE'
  )
```

### 5.2 利润表年度数据查询

```sql
SELECT
  MAX(CASE WHEN item_code='operating_revenue' THEN cumulative_amount END) AS revenue,
  MAX(CASE WHEN item_code='operating_cost' THEN cumulative_amount END) AS cost,
  MAX(CASE WHEN item_code='net_profit' THEN cumulative_amount END) AS net_profit,
  MAX(CASE WHEN item_code='selling_expense' THEN cumulative_amount END) AS selling_expense,
  MAX(CASE WHEN item_code='administrative_expense' THEN cumulative_amount END) AS admin_expense,
  MAX(CASE WHEN item_code='rd_expense' THEN cumulative_amount END) AS rd_expense,
  MAX(CASE WHEN item_code='financial_expense' THEN cumulative_amount END) AS financial_expense
FROM fs_income_statement_item
WHERE taxpayer_id = ?
  AND period_year = ?
  AND period_month = 12
  AND gaap_type = ?
  AND revision_no = (
    SELECT MAX(revision_no) FROM fs_income_statement_item
    WHERE taxpayer_id = ? AND period_year = ? AND period_month = 12 AND gaap_type = ?
  )
```

### 5.3 现金流量年度数据查询

```sql
SELECT
  MAX(CASE WHEN item_code='operating_net_cash' THEN cumulative_amount END) AS operating_cf,
  MAX(CASE WHEN item_code='investing_net_cash' THEN cumulative_amount END) AS investing_cf,
  MAX(CASE WHEN item_code='financing_net_cash' THEN cumulative_amount END) AS financing_cf
FROM fs_cash_flow_item
WHERE taxpayer_id = ?
  AND period_year = ?
  AND period_month = 12
  AND gaap_type = ?
  AND revision_no = (
    SELECT MAX(revision_no) FROM fs_cash_flow_item
    WHERE taxpayer_id = ? AND period_year = ? AND period_month = 12 AND gaap_type = ?
  )
```

### 5.4 年度增值税汇总（一般纳税人）

```sql
SELECT
  SUM(total_tax_payable) AS vat_total,
  SUM(output_tax) AS output_tax_total,
  SUM(input_tax) AS input_tax_total
FROM vw_vat_return_general
WHERE taxpayer_id = ?
  AND period_year = ?
  AND item_type = '一般项目'
  AND time_range = '本月'
```

### 5.5 年度所得税

```sql
SELECT
  actual_tax_payable,
  revenue,
  total_profit,
  taxable_income
FROM vw_eit_annual_main
WHERE taxpayer_id = ?
  AND period_year = ?
ORDER BY revision_no DESC
LIMIT 1
```

### 5.6 发票统计

```sql
-- 销项发票
SELECT COUNT(DISTINCT invoice_pk) AS sales_count,
       SUM(total_amount) AS sales_amount
FROM inv_spec_sales
WHERE taxpayer_id = ? AND period_year = ?

-- 进项发票
SELECT COUNT(DISTINCT invoice_pk) AS purchase_count,
       SUM(total_amount) AS purchase_amount
FROM inv_spec_purchase
WHERE taxpayer_id = ? AND period_year = ?
```

### 5.7 财务指标查询

```sql
SELECT metric_code, metric_name, metric_category,
       metric_value, metric_unit, evaluation_level
FROM financial_metrics_item
WHERE taxpayer_id = ?
  AND period_year = ?
  AND period_month = 12
  AND period_type = 'yearly'
ORDER BY metric_category, metric_code
```
