# 前端交互功能补全实施计划

## Context

根据 `docs/AI智能咨询前端交互技术文档tmp-仅供参考.md` 设计文档，当前 React 前端已实现基础聊天、SSE 流式响应、数据表格/图表/增长分析渲染、历史记录面板等功能，但以下功能缺失或未完整接通：

1. 回答模式（图文/纯数据/简报）未传递到后端，前端未按模式过滤展示
2. 管理消息（批量选择删除）未实现
3. 清空对话缺少确认弹窗和后端同步
4. 导出 PDF 仅调用 window.print()，未实现专用打印窗口
5. 历史记录面板缺少搜索功能
6. 历史记录点击应滚动定位到对话中的对应消息（循环定位），而非替换整个对话
7. 企业选择器未接通到查询流程

---

## Phase 1: 后端接口扩展

### 1.1 扩展 ChatRequest schema
**文件**: `api/schemas.py`
- 添加 `response_mode: str = Field(default="detailed")` — 可选值 detailed/standard/concise
- 添加 `company_id: str = Field(default="")` — 纳税人ID，空字符串表示自动检测

### 1.2 chat 端点透传新参数
**文件**: `api/routes/chat.py`
- `_sse_generator` 签名增加 `response_mode` 和 `company_id` 参数
- `company_id` 非空时，查询 `taxpayer_info` 获取纳税人名称，拼接到 query 前面（利用现有实体预处理的纳税人识别能力）
- `response_mode` 附加到 done 事件的 result dict 中，供前端使用

---

## Phase 2: 状态提升与数据流打通

### 2.1 提升 responseMode 到 App 层
**文件**: `frontend/src/App.jsx`
- 新增 `responseMode` / `setResponseMode` state，传给 ChatArea

**文件**: `frontend/src/components/ChatArea/ChatArea.jsx`
- 接收 `responseMode` prop，传给 ChatInput 和 handleSend

**文件**: `frontend/src/components/ChatInput/ChatInput.jsx`
- 移除本地 `mode` state，改用 props `responseMode` / `onModeChange`

### 2.2 提升 selectedCompanyId 到 App 层
**文件**: `frontend/src/App.jsx`
- 新增 `selectedCompanyId` / `setSelectedCompanyId` state，传给 Header 和 ChatArea

**文件**: `frontend/src/components/Header/Header.jsx`
- 接收 `selectedCompanyId` / `onCompanyChange` props，`<select>` 改为受控组件
- 公司列表加载完成后设置初始值

### 2.3 SSE 和 API 层接通
**文件**: `frontend/src/hooks/useSSE.js`
- `startStream` 签名扩展为 `(query, onEvent, { responseMode, companyId })`

**文件**: `frontend/src/services/api.js`
- `chatStream` 发送 `{ query, response_mode, company_id }`

### 2.4 ChatMessage 按模式过滤展示
**文件**: `frontend/src/components/ChatMessage/ChatMessage.jsx`
- 从 `msg.responseMode` 读取模式
- `concise` 模式：仅显示文本摘要（`dd.summary` 或 `msg.content`），隐藏表格/图表/增长分析
- `standard` 模式：显示表格+增长分析，隐藏图表
- `detailed` 模式：全部显示（当前行为不变）

## Phase 3: 纯前端功能实现

### 3.1 管理消息（批量选择删除）
**文件**: `frontend/src/components/ChatArea/ChatArea.jsx`
- 新增 state: `isSelectionMode` (bool), `selectedIndices` (Set)
- `isSelectionMode` 为 true 时，用选择操作栏替换 ChatInput
- 选择操作栏：显示已选数量 + 取消按钮 + 删除按钮
- 删除操作：`setMessages(prev => prev.filter((_, i) => !selectedIndices.has(i)))`

**文件**: `frontend/src/components/ChatArea/ChatArea.module.css`
- 新增 `.selectionBar` 样式

**文件**: `frontend/src/components/ChatMessage/ChatMessage.jsx`
- 新增 props: `isSelectionMode`, `isSelected`, `onToggleSelect`
- 选择模式下在消息左侧显示 checkbox

**文件**: `frontend/src/components/ChatMessage/ChatMessage.module.css`
- 新增 `.selectWrap`, `.selectCheckbox` 样式

**文件**: `frontend/src/components/ChatInput/ChatInput.jsx`
- bottomRow 中新增"管理消息"按钮，调用 `onToggleSelectionMode` prop

**文件**: `frontend/src/components/ChatInput/ChatInput.module.css`
- 新增 `.manageBtn` 样式

### 3.2 清空对话完善
**文件**: `frontend/src/components/ChatArea/ChatArea.jsx`
- `handleClear` 增加 `window.confirm` 确认弹窗
- 确认后：调用 `cancel()` 中止流式请求 → `setMessages([])` → `deleteHistory([])` 清除后端历史
- 同时重置选择模式状态

### 3.3 导出 PDF
**文件**: `frontend/src/components/ChatArea/ChatArea.jsx`
- 替换 `handleExport`：
  1. 检查 messages 非空
  2. `window.open('', '_blank')` 打开新窗口
  3. 遍历 messages 构建 HTML（用户消息 + AI 消息文本）
  4. 查找当前聊天区域内的 `<canvas>` 元素，用 `toDataURL('image/png')` 转为图片插入 HTML
  5. 写入打印样式（中文字体、表格边框、消息气泡背景色）
  6. `document.write()` 写入内容后调用 `window.print()`

### 3.4 历史记录搜索
**文件**: `frontend/src/components/HistoryPanel/HistoryPanel.jsx`
- 新增 `searchText` state
- 在 header 下方添加搜索输入框
- 按 `item.query.includes(searchText)` 过滤显示
- checkbox 选择改用 `item.timestamp` 作为 key（避免过滤后索引错位）

**文件**: `frontend/src/components/HistoryPanel/HistoryPanel.module.css`
- 新增 `.searchInput` 样式

### 3.5 历史记录点击 → 滚动定位（循环）
**文件**: `frontend/src/App.jsx`
- 新增 `chatAreaRef` (useRef) 传给 ChatArea
- 新增 `historyNavRef` (useRef({})) 记录每个 query 上次定位的索引
- `handleHistoryClick` 改为：
  1. 在当前 messages 中查找所有匹配的用户消息索引
  2. 有匹配：循环定位（最新→上一个→...→回到最新）
  3. 无匹配：回退到当前行为（加载该条 Q&A）

**文件**: `frontend/src/components/ChatArea/ChatArea.jsx`
- 用 `forwardRef` 包装组件
- `useImperativeHandle` 暴露 `scrollToMessage(index)` 方法
- 滚动到目标消息并短暂高亮

**文件**: `frontend/src/components/ChatArea/ChatArea.module.css`
- 新增 `.highlight` 动画样式（2 秒淡出高亮）

---

## Phase 4: 验证

1. 仅发送 `{query}` 到 POST /api/chat，确认后端兼容（Pydantic 默认值生效）
2. 三种模式分别查询，验证 ChatMessage 渲染正确
3. 选择企业后查询，验证返回对应纳税人数据
4. 进入选择模式 → 选择消息 → 删除 → 验证状态一致性
5. 导出 PDF → 验证打印窗口内容完整（含表格和图表图片）
6. 历史搜索 → 输入关键词 → 验证过滤结果
7. 同一问题提问 3 次 → 点击历史记录 → 验证循环定位
