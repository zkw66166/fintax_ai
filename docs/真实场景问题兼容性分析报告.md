# fintax_ai 真实场景问题兼容性分析报告与改造方案

## Context

用户提供了 46 个真实场景问题（单域 18 个 + 跨域 30 个），需要评估现有 NL2SQL 系统能否为每个问题生成准确 SQL。分析发现系统存在若干系统性缺陷，导致绝大多数问题无法正确处理。

---

## 一、系统性缺陷总览

| # | 缺陷 | 影响范围 | 严重度 |
|---|------|---------|--------|
| **G1** | **不支持相对日期**（今年/去年/上个月/上个季度/本月/近N个月等） | 46题中约44题 | 🔴 致命 |
| **G2** | **不支持跨域计算**（差异比较、比率计算、勾稽核对） | 跨域30题中约28题 | 🔴 致命 |
| **G3** | **不支持计算指标**（资产负债率/ROE/毛利率/周转率等） | 约8题 | 🔴 高 |
| **G4** | **季度表达式强制路由到EIT域**（"第一季度"→EIT） | 利润表/CF/VAT按季度查询时 | 🟡 中 |
| **G5** | **不支持枚举月份**（"1月、2月、3月"） | 约3题 | 🟡 中 |
| **G6** | **不支持多年对比**（参数绑定只有单个`:year`） | 约6题 | 🟡 中 |
| **G7** | **子科目查询无同义词**（"管理费用-办公费"等明细科目） | 约4题 | 🟡 低 |

## 四、改造方案

### 优先级 P0：相对日期解析（G1）— 解锁44/46题的前置条件

**改造文件**: `modules/entity_preprocessor.py`

**方案**: 在 `detect_entities()` 的期间提取逻辑前，增加相对日期→绝对日期的预处理函数 `_resolve_relative_dates(query, today)`：

```
支持的相对表达式 → 转换规则：
├── "今年"/"本年"           → 当前年份 (2026)
├── "去年"/"上年"           → 当前年份-1 (2025)
├── "前年"                  → 当前年份-2 (2024)
├── "本月"/"这个月"         → 当前年+月 (2026年2月)
├── "上个月"/"上月"         → 当前月-1 (2026年1月)
├── "上个季度"              → 当前季度-1 → 展开为月份范围
├── "今年上半年"            → 当前年1月到6月
├── "今年前N个月"           → 当前年1月到N月
├── "最近N个月"/"近N个月"   → 从当前月往前推N个月
├── "最近N个季度末"         → 计算最近N个季度末月份列表
├── "过去N个纳税年度"       → 当前年-N 到 当前年-1
├── "近N年"/"最近N年"       → 当前年-N+1 到 当前年
├── "去年底"/"年底"         → 去年12月 / 当前年12月
```

实现方式：正则替换，将相对表达式替换为绝对日期字符串后，再走现有的绝对日期解析逻辑。

### 优先级 P1：跨域计算引擎（G2）— 解锁28/30跨域题

**改造文件**: `mvp_pipeline.py`, `modules/sql_writer.py`, `prompts/stage2_cross_domain.txt`

**方案**: 将跨域查询从"单次UNION ALL"改为"多次查询 + Python端计算"：

```
当前流程:
  entity → intent → constraint → 单SQL(UNION ALL) → 执行

改造后流程:
  entity → intent → 拆分为N个子查询(每域一个)
       ↓
  对每个子域: constraint → SQL生成 → 审计 → 执行 → 子结果
       ↓
  Python端合并/计算/比较 → 最终结果
```

关键改造点：
1. `mvp_pipeline.py` 新增 `_run_cross_domain_pipeline()` 函数，循环每个子域独立执行
2. 新增 `modules/cross_domain_calculator.py`，支持：
   - 差异计算（A - B）
   - 比较判断（A == B? 差异多少?）
   - 比率计算（A / B * 100%）
   - 勾稽核对（多值一致性检查）
3. Stage 1 intent 增加 `cross_domain_operation` 字段（compare/ratio/reconcile/list）
4. 保留 UNION ALL 模式用于简单的"列举型"跨域查询

### 优先级 P1：计算指标支持（G3）— 解锁8题

**改造文件**: `modules/metric_calculator.py`（新建）, `modules/intent_parser.py`

**方案**: 建立指标公式注册表：

```python
METRIC_FORMULAS = {
    '资产负债率': {
        'formula': 'total_liabilities / total_assets * 100',
        'sources': {
            'total_liabilities': {'domain': 'balance_sheet', 'column': 'LIABILITIES_TOTAL_end'},
            'total_assets': {'domain': 'balance_sheet', 'column': 'TOTAL_ASSETS_end'}
        }
    },
    '净资产收益率': {  # ROE
        'formula': 'net_profit / average_equity * 100',
        'sources': {
            'net_profit': {'domain': 'profit', 'column': 'net_profit'},
            'average_equity': {'domain': 'balance_sheet', 'column': '(EQUITY_begin + EQUITY_end) / 2'}
        }
    },
    '毛利率': {
        'formula': '(revenue - cost) / revenue * 100',
        'sources': {
            'revenue': {'domain': 'profit', 'column': 'operating_revenue'},
            'cost': {'domain': 'profit', 'column': 'operating_cost'}
        }
    },
    '总资产周转率': {
        'formula': 'revenue / average_total_assets',
        'sources': {
            'revenue': {'domain': 'profit', 'column': 'operating_revenue'},
            'average_total_assets': {'domain': 'balance_sheet', 'column': '(TOTAL_ASSETS_begin + TOTAL_ASSETS_end) / 2'}
        }
    },
}
```

Stage 1 检测到计算指标时，自动拆解为子查询 + Python端公式计算。

### 优先级 P2：季度路由修复（G4）

**改造文件**: `modules/entity_preprocessor.py` (line 364-365)

**方案**: 移除"季度查询默认EIT"的硬编码逻辑。改为：
- 如果已有域检测结果（如利润表/CF），保留该域，仅记录 `period_quarter` 用于月份展开
- 仅当无其他域信号时，才默认EIT
- 季度→月份展开：Q1→1,2,3月; Q2→4,5,6月; Q3→7,8,9月; Q4→10,11,12月

### 优先级 P2：多年对比与枚举月份（G5+G6）

**改造文件**: `modules/entity_preprocessor.py`, `mvp_pipeline.py`

**方案**:
- 枚举月份："1月、2月、3月" → 解析为 `period_months: [1, 2, 3]`
- 多年对比："去年和今年" → 解析为 `period_years: [2025, 2026]`
- `_build_params()` 支持列表参数，SQL使用 `IN (:year1, :year2)` 或动态生成

### 优先级 P3：子科目同义词（G7）

**改造文件**: `database/seed_data.py`（account_synonyms表）

**方案**: 为常见子科目添加同义词映射，如：
- "管理费用-办公费" → account_name LIKE '管理费用%办公费%'
- "应交税费-应交增值税" → account_name = '应交增值税'
- "应交税费-未交增值税" → account_name = '未交增值税'

同时在 Stage 2 account_balance prompt 中增加子科目查询模式说明。
